Abstract

These release notes define the operation and usage of installing Oracle E-Business Suite (EBS) 12.2.5 using Chef Client version 12.6 on the AIX® 7.1 platform.

Not supported

The Oracle EBS product supports multiple environments and configuration. This cookbook does not support the following options within Oracle EBS:

§  Does not support an Oracle RAC installation of the database. This version only supports the single database option.

§  Does not support multi-tier installation. This cookbook installs the database, application server and WebLogic server on a single operating system.

§  Does not install the production version of the database. This cookbook installs the VIS demonstration database. While it may be easy to change the options to the production database, it currently has not been tested or provided an option within the code.

§  Does not support other operating systems. There are no current plans to support a different operating system.

§  Has not been tested on AIX 6.1. 

Requirements

Disk space requirements

The database file system, by default, /d01 is 400GB in size. It houses the binaries provided by Oracle, temporary data space and the Oracle Database files. The Oracle Database files consumes roughly 200GB of space when installed.

The application file system, /applmgr, is 100GB in size, defined within this cookbook. Actual usage space is less than 100GB when installed.

The /opt file system is expanded to 10GB within the cookbook. The /tmp file system is expanded to 10 GB as well.  /opt is where the chef installation occurs. An Oracle EBS install requires /tmp to be larger than the default on the AIX 7.1 operating system.

The oracle_ebs cookbook will create the volume group, ebsvg01, and the file systems, /d01 and /applmgr, if given a set of disk drives that are available. See the default.rb attributes file for details. Otherwise, the cookbook expect to see /d01 and /applmgr created beforehand.

 

#-------------------------------------------------------

# Volume Group Attributes – used to create the file systems

#-------------------------------------------------------

#

default[:ebs][:vg][:ppsiz]        = 128

default[:ebs][:vg][:pp_per_gig]   = 1024/node[:ebs][:vg][:ppsiz]

default[:ebs][:vg][:vgname]       = 'ebsvg01'

default[:ebs][:vg][:db_fs_siz]             = '398' # in GIGS

default[:ebs][:vg][:app_fs_siz]            = '198' # in GIGS

 

#-------------------------------------------------------

# File System Attributes

#-------------------------------------------------------

#

default[:ebs][:vg][:db_fs_nam]             = '/d01'

default[:ebs][:vg][:app_fs_nam]            = '/applmgr'

default[:ebs][:vg][:tmp_fs_siz]    = '10' # in GIGS

default[:ebs][:vg][:opt_fs_siz]    = '10' # in GIGS

default[:ebs][:vg][:fsopts] = "-Ayes -prw -a agblksize=4096 -a logsize=1024 -a isnapshot=no"

                              

 

Operating system requirements

The system used for testing the Oracle EBS installation was an IBM Power Systems™ server with two POWER8™ processors and 32GB of memory. The cookbook will modify the kernel parameter maxuproc and kernel parameter iocp, which are required by the Oracle EBS installation.

The paper, Porting Chef Cookbooks to AIX, provides additional useful information regarding operating system requirements, and will not be duplicated here.

Oracle E-Business Suite

This cookbook installs version 12.2.2.5 of the standard Oracle E-Business Suite product. Most likely, the version of Oracle EBS that you wish to install would be either a slightly newer version or a version on another platform. The steps, and documents I used to create the Oracle EBS Cookbook should be similar to your example. As such, it should be fairly easy to port the oracle_ebs Cookbook to other Oracle EBS versions.

The following Oracle documents were used during the process of creating the Oracle EBS installation cookbook. These documents are available from the My Oracle Support website (a valid userid and password are required to access that site).

Doc ID: 1983050.1 was the main document for driving the installation instructions. The other two documents, in bold, were also largely important to the install. The remaining documents were occasionally used as required. I also kept the update date, in the case that these documents are updated, then the installation of Oracle EBS for newer patches and release will likely require re-testing and porting.

Staging area

The installation of the Oracle EBS Cookbook is highly dependent on the staging area. In this cookbook, the staging area is nfs mounted. There are two attributes within the default.rb file that determine the mount point and mounting within the Oracle EBS Cookbook.

The nfs mount must be set up for the Oracle EBS Cookbook to remotely connect from the target installation machine.

default[:ebs][:stage][:nfsmount]       = /ebstage

default[:ebs][:stage][:nfshost]        = <CHANGE_ME>

default[:ebs][:stage][:dir]            = /ebstage/stage

default[:ebs][:stage][:zips]           = /ebstage/zips

Figure 1. Attributes within the attributes/default.rb file

The directory structure is well documented in the “Oracle E-Business Suite Installation Guide: Using Rapid Install” document. You start with unzipping the startCD zip files. Following the process for building the staging area, we end up with. /ebstage/stage:

DBMS/              EBSInstallMedia/   TechInstallMedia/  TechPatches/       startCD/

The other significant directory is the zips directory. It contains all the patches files. Appendix A contains the list of patch files found within the /ebstage/zips directory.

AIX

AIX is an open, standards-based operating system that conforms to The Open Group's Single UNIX® Specification Version 3. It provides fully integrated support for 32- and 64-bit applications. The AIX operating system provides binary compatible support for the entire IBM UNIX product line including IBM Power™ Systems, System p™, System i™, pSeries®, and iSeries™ servers[PMM1] . AIX also supports qualified systems offered by hardware vendors participating in the AIX Multiple Vendor Program. So, as you move to newer versions of the AIX operating system, its excellent history of binary compatibility provides confidence that your critical applications will continue to run.

Chef knife bootstrap syntax for Oracle EBS Cookbook

The Chef knife bootstrap command has many options. The syntax of the Chef knife bootstrap command below provides the options used when running the Oracle EBS Cookbook in our testing. Many more options are available and can be found at the knife bootstrap Chef documentation.

knife bootstrap <TARGETNODE> -x root -P <root_password> -N <TARGETNODE> -r role[role_ebs]

The TargetNode represents the machine in which you want to provision for Chef. In our case, the TargetNode would be an AIX 6.1 or 7.1 machine or LPAR. The –x option defines the SSH user name on the TargetNode. This is typically the root user. The –P option specifies the SSH password for the SSH user name on the TargetNode. The –r option describes the Chef run-list, which in our case is the role ‘mach51’ role cookbook.

Name "role_ebs"

Description "Role applied to Oracle E-Business Suite on AIX 7.1."

 

run_list 'recipe[ebs]'

 

override_attributes :ebs_appuser   => 'applmgr',

   :ebs_dbuser                     => 'oraprod',

   :ebs_groupid                    => '1000',

   :ebs_group                      => 'oinstall',

 :ebs => {

   :app   =>  { :usr => { :uid     => 2000 } },

   :db    =>  { :usr => { :uid     => 3000 } },

   :stage =>  { :nfshost           => '<nfshostname>',

                :nfsmount          => '/ebstage'},

   :vg    =>  { :app_fs_nam        => '/applmgr',

                :app_fs_siz        => 198, # GB

                :db_fs_nam         => '/d01',

                :db_fs_siz         => 398, # GB

                :sashosts          => [ 'TARGETNODE' ],

                :swapspace         => 16384,

                :vgname            => 'ebsvg01',

                :drives            => { 'TARGETNODE' => [ 'hdiskX', 'hdiskY',                    'hdiskZ' ] },

             },

  }

As mentioned earlier, the Oracle EBS cookbook will create the file systems /applmgr and /d01. It requires the unused hdisks to provide the space for a new volume group. The Oracle EBS cookbook assumes that we are provisioning a new OS, and that we have spare drives. If this is not the case, then if the volume group already exists, creation will be bypassed.

The mach51 role file is an example of overriding the default attributes for installing in different environments. The key areas that may be different for your installation are:

§  User names and UID’s

§  The remote mount file system details.

§  The size of swap space (it is taken from rootvg volume group)

§  And file system and volume group names.

Cookbook structure

The logical flow of the Oracle EBS Cookbook is as follows, and is based on the Doc ID: 1983050.1 Oracle EBS Release 12.2.5 Readme.html file:

1.      Install the OS related environment. This includes kernel parameters, file system and users and groups. Also checks for required software.

2.      The stage directory is expected to be mounted and created, and the Oracle EBS installation begins with the rapidwiz utility, installing the base Oracle EBS 12.2.0 software. The EBS 12.2.0 base installs version 11.2.0.3 of the Oracle Database.

3.      The 11.2.0.4 Oracle Database is installed. 11.2.0.4 Oracle Database patches are installed.

4.      The Oracle Database utility ‘dbua’ is used to migrate the Oracle EBS 11.2.0.3 database to 11.2.0.4

5.      The additional patches as specified by Doc ID: ‘Oracle E-Business Suite Release 12.2: Consolidated List of Patches and Technology Bug Fixes’ are installed.

6.      The Oracle EBS AD and TXK patches are installed.

7.      The Oracle EBS Upgrade patch to upgrade the Oracle EBS software to 12.2.5 is applied.

8.      Final patches, cleanup and clone is applied.

9.      Post installation patches are applied.
